Développement d'un cyberfolio en équipe : retour d'expérience

Dans le cadre d'un projet pédagogique DevSecOps, j’ai participé à la création d’un cyberfolio en équipe (Back to the Stack). L’objectif : concevoir et développer un site web full-stack fonctionnel pour présenter une équipe de développeurs, en approchant toutes les étapes du cycle de la maquette au back-end.

Dépôt du projet

Outils et organisation

Nous avons utilisé :

  • Discord pour la communication.
  • GitHub pour le versioning (malgré quelques conflits résolus collectivement).
  • Trello pour la gestion des tâches en Kanban.

Conception : poser les bases

Avant de commencer à coder, nous avons réalisé :

  • Une arborescence du site pour définir les pages et leurs contenus.
  • Une maquette visuelle sur Canva, pour fixer une identité graphique (palette, structure, cohérence CSS).

Maquettes

Front-end

Nous avons utilisé HTML, CSS et Javascript pour donner vie à notre cyberfolio.
Exemple de fonctionnalités:

  • Image interactive: sur la page « Notre équipe », chaque zone cliquable renvoie vers la présentation d’un membre.
  • Menu responsive: une barre de navigation transformée en menu hamburger sur mobile, gérée avec des media queries et du JavaScript
  • Mode jour/nuit: un switch qui change la feuille de style (light/dark) via localStorage.

Back-end

Côté serveur, nous avons travaillé en PHP/MySQL.

  • Pages dynamiques: la page membre.php affiche automatiquement les informations d’un membre (photo, email, compétences) à partir de la BDD.
  • Base de données relationnelle: tables membres, competences et membre_competences qui relie les deux précédentes grâce aux id.
  • Formulaire de recherche: l’utilisateur peut sélectionner une compétence, et le site renvoie les membres qui la possèdent (requêtes préparées SQL avec bind_param).
  • Système d’authentification: inscription et connexion via une table users, avec mots de passe hachés en MD5 (approche simple mais qui ouvre la discussion sur la sécurité des hash).

Ce que ce projet m'a apporté

Au delà des compétences techniques (HTML/CSS, PHP/MySQL, Git), ce projet m’a permis de créer :

  • Une base de données relationnelle: tables membres, competences et membre_competences qui relie les deux précédentes grâce aux id.
